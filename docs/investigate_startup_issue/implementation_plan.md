# アプリケーション起動不可の調査計画

## 概要

現在、`stt_daemon.py` は実行されていますが、GUIが起動しておらず、アプリケーションが正常に動作していない可能性があります。この計画では、原因の特定と修正手順をまとめます。

## 現状の分析

- `stt_daemon.py` (PID 2048) はバックグラウンドで動作中。
- `gui.py` は動作していない。
- `core.log` には "Terminated" の記録のみ。

## 調査・修正ステップ

1. **GUIの手動起動テスト**: `python3 gui.py` を実行し、エラーメッセージを確認します。
2. **デーモンの状態確認**: デーモンが正常に応答しているか、ログ出力を確認します。
3. **依存関係の確認**: `requirements.txt` に基づき、必要なライブラリが不足していないか確認します。
4. **VRAM/GPU状態の確認**: `nvidia-smi` 等でGPUリソースの競合がないか確認します。

## 検証計画

### 手動検証

- `start_gui.sh` または `python3 gui.py` を実行してGUIが表示されることを確認します。
- ホットキー（Ctrl+Shift+Space）を押して、音声入力が開始されるか確認します。
