# Task: 音声入力ソフトウェアの提案と既存システムのログ・堅牢性強化

## 目的
1. 手間を省きたいユーザーへ向け、Linuxで動作するおすすめのローカル音声入力（STT）ソフトウェアを提案する。
2. 現在の `speech_to_text` プロジェクトにおいて、「まともに動かない」原因を追及できるようにするため、ログ出力機構の導入とエラーハンドリング（堅牢性）の強化プランを立案する。

## 背景
- 現在のシステムは `stt_daemon.py` や `stt_worker_unified.py` から構成されているが、エラー時に出力が単なる `print` メッセージに留まっており、スタックトレースが追いにくい。
- ユーザーはすでにトラブルシューティングに疲弊しており、代替となる既製品のローカル音声入力ソフトの紹介も求めている。

## 実施事項
1. 代替ソフトウェア（Speech Note, Nerd Dictation, BlabbyAI for Linux など）の調査結果をユーザーに提示。
2. `speech_to_text` プロジェクトの現状分析から、Python標準の `logging` モジュールを導入して詳細なファイルログを残す改修案を策定。
3. これらの情報をドキュメント化し、チャット上で回答・提案を行う。
