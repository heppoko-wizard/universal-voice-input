# Walkthrough: 音声入力ソフトウェアの提案と既存システムのログ・堅牢性強化

## これまでに実施したこと

1. **Linux向けローカル音声入力ソフトウェア（STT）の調査**
   - ユーザーからの依頼「Linuxで動くおすすめのローカル音声入力のソフトウェアを教えて」に基づき、いくつか代替案を調べました。
   - `Speech Note` (Flathub), `Nerd Dictation` (ターミナル＋Vosk), `Vocalinux` (Wayland/X11対応)をピックアップし、これらがユーザーの要件を満たすソフトウェアであることを確認しました。特に、デスクトップ統合が進んでいる Speech Note を最もお勧めします。

2. **既存システム (`speech_to_text`) のログ・堅牢性強化プランの作成**
   - 原因追及のため、`stt_daemon.py` 等の既存コードを確認しました。現在、エラーログが `print()` コマンドに依存しており、致命的な例外が発生しても詳しいスタックトレースが残らない仕様になっていることが判明しました。
   - 今後も自作ツールを併用したい場合のために、Python標準の `logging` モジュールを使ったファイル出力の導入、およびプロセス監視の再設計（例外情報の保持とクラッシュ時の再起動の徹底）を行う実装計画（Implementation Plan）を策定しました。
